## Objective

Comprehensive enhancement of the NoeAgent framework: merge persistent subagent design into RFC-0005, move NoeAgent to top-level module, create tool-vs-subagent implementation guide, and build integration benchmark suite.

## Completed

1. **Module restructuring**: Moved `noesium/agents/noe/` to `noesium/noe/` as top-level module
   - Updated all imports across 15+ files (tests, examples, docs, config)
   - Created backward-compatibility shim at old location
   - Updated `pyproject.toml` entry point from `neo-agent` to `noe-agent`

2. **RFC-0005 merge**: Integrated RFC-0005-Patch1 (Persistent CLI Subagent Daemon) into RFC-0005
   - Added CapabilityType taxonomy (§4.2): InProcessTool, RemoteTool, Subagent, ExternalCliSubagent
   - Added Persistent Subagent Architecture (§10): adapter interface, communication protocol, session multiplexing
   - Added Subagent Lifecycle Model (§11): state machine, failure recovery, scheduling
   - Added Tool vs Subagent comparison (§16): dimensional analysis table
   - All references made abstract (no NoeAgent-specific mentions)
   - Deleted RFC-0005-patch1.md

3. **NoeAgent impl guide polish** (`docs/impl/noe-agent-impl.md`):
   - Enhanced tool call design: arg pre-validation, max_tool_calls enforcement, error recovery
   - Added Claude CLI subagent daemon integration (§5.10): ExternalCliAdapter, CliSubagentConfig
   - Added intelligent auto-planning for subagent spawning (§5.11): decision heuristics, execution hints
   - Added subagent cleanup lifecycle (§5.4.3)
   - Added implementation gap analysis (§9) with prioritized enhancement plan
   - Updated all module paths to `noesium/noe/`

4. **New impl guide**: Created `docs/impl/noe-agent-tool-vs-subagent.md`
   - Ontological distinction (capability vs cognition extension)
   - Control model, state/memory, temporal scope, failure semantics
   - NoeAgent decision matrix with routing heuristics
   - Implementation mapping to AgentAction schema and graph nodes
   - Anti-patterns and cost/performance considerations

5. **Integration benchmark suite** (`tests/agents/benchmark_noe_agent.py`):
   - Per-toolkit tests for all 18 toolkits (web_search, bash, python_executor, etc.)
   - Subagent orchestration tests (spawn, parallel, depth limit)
   - Error resilience tests (validation error regression from testerror.md, permission denied)
   - End-to-end scenarios (project name uniqueness research, planning/reflection cycle)
   - Benchmark infrastructure: BenchmarkCollector, progress event lifecycle verification
