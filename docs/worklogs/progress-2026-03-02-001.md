## Objective

Address four concrete issues identified from test runs: subagent tool registration bug, default path pollution, terse TUI progress display, and broken test methods.

## Completed

- Changed default `session_log_dir` from `.noe_sessions` to `~/.noe_agent/sessions` and `memu_memory_dir` from `.noe_memory` to `~/.noe_agent/memory` in `config.py`
- Added `_register_subagent_tools()` method in `agent.py` that registers `spawn_subagent` as a callable tool via `FunctionAdapter` when `enable_subagents=True`
- Added fallback handler in `tool_node` (nodes.py) that intercepts `tool_name="subagent"` calls and redirects them to `spawn_subagent`, preventing "Tool 'subagent' not registered" errors
- Enhanced `astream_progress()` TOOL_START summaries to include brief arg values (e.g., `Using web_search(query="noesium domain")`)
- Enhanced TOOL_END summaries to show first line of result instead of generic "Completed"
- Enhanced SUBAGENT_START summaries to include the task description
- Updated `_activity_line()` in tui.py to consume the richer summaries
- Removed obsolete `test_render_tool_call_panel_*` test methods and replaced with `test_activity_line_*` tests
- Updated `TestStreamEvents` to match new `ProgressEvent.model_dump()` key format (`plan.created`, `final.answer`, `tool.start`)
- Added `test_session_lifecycle_events` test for session.start/session.end
- Added `/session` assertion to `test_slash_commands_constant`
- Updated `noe-agent-impl.md` with new default paths, subagent-as-tool documentation, and revision history
