# NoeAgent: Timestamp and Adaptive Final Report

## Objective

- Make agent prompt timestamp explicit so the model can detect future-dated search results (e.g. 2026 sources when "current" is 2024).
- Replace the single-style final answer with an adaptive final report: simple (brief), complex (detailed), deep research (full report).

## Completed

- Optimized `_get_current_datetime()` to return "Today's date: YYYY-MM-DD. Current time (UTC): HH:MM:SS." for clear comparison with source dates.
- Added in `agent_system.md` a bullet: when using web search, treat sources dated after today as unreliable and note the current date to the user.
- Redesigned `finalize.md`: three tiers (simple / complex / deep research), optional `current_datetime`, and date-consistency guidance for synthesis.
- Passed `current_datetime` into the finalize node so the final report can reference it and warn on future-dated sources.
- Updated `prompts/README.md` to describe finalize's adaptive report and optional variable.

- `noesium/noeagent/prompts/finalize.md` — Adaptive report depth (simple/complex/deep), context with `current_datetime`, date-consistency and quality rules.
- `noesium/noeagent/prompts/README.md` — Finalize row updated for adaptive synthesis and optional `current_datetime`.
