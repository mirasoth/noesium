## Objective

Per RFC-1006 (Extensible Subagent Interface), polish the subagent implementation:
1. Make `noesium/core/agent/subagent/` fully independent of NoeAgent
2. Remove NoeAgent-specific adapters from core; adapt NoeAgent to use new interfaces directly
3. Add comprehensive unit and integration tests

## Completed

### Core Subagent Framework (`noesium/core/agent/subagent/`)
- Updated RFC references from RFC-9003 → RFC-1006 across all files
- Enhanced `SubagentProvider` in `protocol.py`:
  - Added `from_instance()` classmethod for pre-built runtime instances
  - Added `from_factory()` classmethod for factory-callable construction
  - Changed `_runtime_class` → `_runtime_factory` for uniform interface
- Fixed `BaseSubagentRuntime.descriptor` property to fall back to `get_descriptor()` when no instance `_descriptor` is set
- Fixed `BaseSubagentRuntime.resume()` to be an async generator (was a coroutine, incompatible with `async for`)
- Deleted `adapters.py` (contained NoeAgent-specific `InProcSubagentAdapter`, `BuiltinSubagentAdapter`, `CliSubagentAdapter`)
- Updated `__init__.py`: removed adapter exports, updated module docstring

### NoeAgent-Specific Runtimes (`noesium/noeagent/subagent_runtimes.py`)
- Created new file with three `BaseSubagentRuntime` implementations:
  - `NoeChildSubagentRuntime`: wraps child NoeAgent, translates ProgressEvents → SubagentProgressEvents
  - `NoeBuiltinSubagentRuntime`: lazy factory for browser_use/tacitus, streams via astream_progress
  - `NoeCliSubagentRuntime`: wraps ExternalCliAdapter for oneshot CLI execution

### NoeAgent Integration (`noesium/noeagent/`)
- `agent.py`:
  - Added `_subagent_manager: SubagentManager | None` field
  - `initialize()` creates SubagentManager with configurable timeout/concurrency
  - `_setup_builtin_subagents()`: registers `NoeBuiltinSubagentRuntime` in SubagentManager (not registry)
  - `_setup_external_subagents()`: registers `NoeCliSubagentRuntime` in SubagentManager
  - `spawn_subagent()`: registers `NoeChildSubagentRuntime` in SubagentManager
  - `interact_with_subagent()`: delegates to `invoke_subagent()`
  - Added `invoke_subagent()`: unified invocation via SubagentManager, fires progress callbacks
  - Added `_subagent_event_to_progress()`: translates SubagentProgressEvent → ProgressEvent
  - `invoke_subagent()` now raises `RuntimeError` on SUBAGENT_ERROR events (proper error propagation)
  - `execute_builtin_subagent_streaming()`: new signature `(subagent_id, message)`, delegates to `invoke_subagent()`
- `nodes.py`:
  - `invoke_builtin`: uses `agent.invoke_subagent()` via SubagentManager
  - `invoke_cli`: uses `agent.invoke_subagent()` with fallback to `_cli_adapter.execute_oneshot()`
  - Fixed fallback to pass `allowed_tools` and `skip_permissions` kwargs from SubagentAction
- `commands.py`: uses `agent.invoke_subagent()` instead of registry-based provider lookup
