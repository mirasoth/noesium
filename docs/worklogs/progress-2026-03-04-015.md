# Implement /browser and Tacitus Plan Fixes

## Objective
Implement the recommended fixes from the plan "Why /browser Does Not Trigger browser_use Agent": config merge for Tacitus requires_explicit_command, defensive logging in astream_progress, and documentation for preserving full user input in backend task description.

## Completed
- Added `_builtin_from_global_config()` in noeagent config to merge global `subagents.builtin` with `DEFAULT_AGENT_SUBAGENTS` by name so Tacitus keeps `requires_explicit_command=True` when loading from file.
- Replaced inline builtin construction in `NoeConfig.from_global_config()` with a call to `_builtin_from_global_config(global_config.subagents.builtin)`.
- Added defensive logging in `NoeAgent.astream_progress`: log when an inline command is detected (INFO) and when the main graph will run (DEBUG).
- Documented task description contract: `TaskCreate.description` field description and a comment in `TaskOrchestrator._execute_task` stating that full user input (including /browser, /research) must be preserved for correct routing.

## Files Changed
- `noesium/noeagent/config.py` - Added `_builtin_from_global_config()`, use it in `from_global_config()` for builtin list.
- `noesium/noeagent/agent.py` - Log inline command vs main graph in `astream_progress`.
- `coder-backend/src/voyager/models/task.py` - Added Field description on `TaskCreate.description` to preserve slash commands.
- `coder-backend/src/voyager/services/task_orchestrator.py` - Comment that `task.description` must be full user input for inline command routing.

## Tests
- No new tests run. Existing linter: no new errors in modified files (agent.py has pre-existing basedpyright warnings).

## Notes
- Tacitus auto-routing was caused by core `AgentSubagentConfig` lacking `requires_explicit_command`; when loading from global config the attribute was missing so it defaulted to False. Merging with defaults by name fixes this without changing core config schema.
- Backend and frontend already pass description through unchanged; the change is documentation only for future maintainers.
