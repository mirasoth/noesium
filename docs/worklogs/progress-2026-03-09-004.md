## Objective

Fix the `name 'AgentState' is not defined` error that occurred when NoeAgent tried to compile its LangGraph state machine.

## Root Cause Analysis

The error occurred due to a Python limitation with **forward reference annotations** and **local imports**:

1. `builder.py` uses `from __future__ import annotations` which defers type annotation evaluation
2. `AgentState` was imported **inside** `build_agent_graph()` function (local scope)
3. Inner routing functions like `_route_exec(state: "AgentState")` had type annotations
4. LangGraph's `add_conditional_edges()` calls `typing.get_type_hints()` on routing functions
5. `get_type_hints()` looks in the function's **global namespace** (module level), not local scope
6. Since `AgentState` was only in local scope → **NameError**

## Solution

Moved the `AgentState` and `AskState` imports from inside functions to module level in `builder.py`, ensuring they're available in the module's globals when `get_type_hints()` is called.
