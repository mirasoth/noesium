# NoeAgent NOESIUM_HOME Override and Browser Headless Fix

## Objective
Implement RFC-1007 requirement for NoeAgent to override the default `NOESIUM_HOME` so the framework uses `~/.noeagent` instead of `~/.noesium`. Also fix the `BROWSER_USE_HEADLESS=false` environment variable parsing bug.

## Completed
- Added `set_noesium_home()` function to `noesium/core/consts.py` to allow runtime override of `NOESIUM_HOME`
- Modified `noesium/__init__.py` to call `set_noesium_home()` BEFORE any other imports (critical for proper initialization)
- Modified `noesium/noeagent/config.py` to also call `set_noesium_home()` as fallback for direct imports
- Changed `noesium/core/config.py` to use `Field(default_factory=...)` for path defaults (ensures late binding)
- Removed unused global constants from `noesium/core/consts.py` (`NOESIUM_LOGS_DIR`, `NOESIUM_MEMORY_DIR`, etc.)
- Exported `set_noesium_home` in `noesium/core/__init__.py` for public API access
- Fixed `BROWSER_USE_HEADLESS` parsing in `noesium/noeagent/agent.py` to properly handle `false` values

## Files Changed
- `noesium/core/consts.py` - Added `set_noesium_home()` function; removed unused directory constants
- `noesium/core/config.py` - Changed path defaults to use `Field(default_factory=...)` for late binding
- `noesium/__init__.py` - Added `set_noesium_home()` call BEFORE all other imports
- `noesium/noeagent/config.py` - Import and call `set_noesium_home(_NOEAGENT_HOME)` on module load
- `noesium/core/__init__.py` - Export `set_noesium_home` in `__all__`
- `noesium/noeagent/agent.py` - Fixed env var parsing for `BROWSER_USE_HEADLESS`

## Tests

## Notes
- The `set_noesium_home()` function must be called BEFORE any imports that use `NOESIUM_HOME`
- The key fix was moving the call to `noesium/__init__.py` since it imports from `noesium.core.config`
- Using `Field(default_factory=...)` ensures paths are computed at instantiation time, not class definition time
- For core-only usage without NoeAgent, users should import directly from `noesium.core`
