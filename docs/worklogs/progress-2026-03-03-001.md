## Notes

### Event Flow Architecture

```
NoeAgent.astream_progress()
    ‚îú‚îÄ‚îÄ ProgressEvent (SESSION_START)
    ‚îú‚îÄ‚îÄ ProgressEvent (PLAN_CREATED)
    ‚îú‚îÄ‚îÄ BuiltInAgentCapabilityProvider.invoke_streaming()
    ‚îÇ   ‚îî‚îÄ‚îÄ BrowserUseAgent.astream_progress()
    ‚îÇ       ‚îú‚îÄ‚îÄ ProgressEvent (SESSION_START) ‚Üí wrapped as skip
    ‚îÇ       ‚îú‚îÄ‚îÄ ProgressEvent (THINKING) ‚Üí wrapped as SUBAGENT_PROGRESS
    ‚îÇ       ‚îú‚îÄ‚îÄ ProgressEvent (TOOL_START) ‚Üí wrapped as SUBAGENT_PROGRESS
    ‚îÇ       ‚îú‚îÄ‚îÄ ProgressEvent (FINAL_ANSWER) ‚Üí wrapped as SUBAGENT_PROGRESS
    ‚îÇ       ‚îî‚îÄ‚îÄ ProgressEvent (SESSION_END) ‚Üí wrapped as skip
    ‚îÇ   ‚îî‚îÄ‚îÄ ProgressEvent (SUBAGENT_END)
    ‚îî‚îÄ‚îÄ ProgressEvent (SESSION_END)
```

### Key Design Decisions

1. **Event Wrapping**: SESSION_START/SESSION_END from child agents are skipped to avoid nesting; only meaningful progress events are wrapped as SUBAGENT_PROGRESS.

2. **Metadata Preservation**: Child event type and agent type are preserved in metadata for TUI to render agent-specific icons.

3. **Queue Bridge**: `asyncio.Queue` bridges callback-style event emission with generator-style yielding in `astream_progress()`.

4. **Backward Compatibility**: Non-streaming agents fall back to blocking `invoke()` with simple START/END events.
