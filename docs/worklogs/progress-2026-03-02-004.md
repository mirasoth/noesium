## Completed

### `planning_model` Wiring (P2)
- `TaskPlanner.__init__` now accepts optional `planning_llm` parameter
- `NoeAgent.initialize()` creates a separate LLM client when `NoeConfig.planning_model` is set
- Falls back to the main LLM if the planning LLM fails to initialize

### Execution Hints in Planner (P2)
- `PLANNING_PROMPT` updated to request `execution_hint` per step with available modes (tool/subagent/cli_subagent/auto)
- `REVISE_PLAN_PROMPT` updated to also include execution hints
- `TaskPlanner._parse_steps()` now extracts `execution_hint` from LLM JSON output with validation
- `TaskPlanner` receives `cli_subagent_names` to include available CLI subagents in the planning prompt

### Execution Hint in LLM Prompt (P2)
- `AGENT_SYSTEM_PROMPT` now includes `{execution_hint}` section
- `execute_step_node` reads `current_step.execution_hint` and maps it to human-readable guidance text
- This biases the LLM toward the recommended execution strategy per step

### STEP_COMPLETE Event Emission (P2)
- `astream_progress()` now tracks `_prev_step_index` and detects when `plan.advance()` has completed steps
- Emits `ProgressEventType.STEP_COMPLETE` events with `step_index` and `step_desc` for each completed step
- Fills the gap in the event lifecycle: `STEP_START → ... → STEP_COMPLETE → STEP_START → ...`

### Subagent Cleanup (P2)
- New `_cleanup_subagents()` method clears `_subagents` dict and calls `cli_adapter.terminate_all()`
- Called in `astream_progress()` finally block, ensuring cleanup on both success and error paths

### ExternalCliAdapter (P2)
- New module `noesium/noe/cli_adapter.py` implementing RFC-0005 §10
- `SubagentHandle` Pydantic model: tracks name, state (CREATED/RUNNING/BUSY/IDLE/TERMINATED), pid, session_id
- `ExternalCliAdapter` class with full lifecycle:
  - `spawn(name, message)` / `spawn_from_config(config, message)` — start daemon via `asyncio.create_subprocess_exec`
  - `interact(name, message)` — send JSON line to stdin, read response from stdout with timeout
  - `health_check(name)` — check if process is alive
  - `restart(name)` — terminate then re-spawn
  - `terminate(name)` / `terminate_all()` — graceful SIGTERM then SIGKILL after 5s
  - `get_status()` — status summary of all managed daemons
- Communication protocol: line-delimited JSON over stdin/stdout

### CLI Adapter Wiring (P2)
- `NoeAgent._setup_cli_subagents()` initializes the adapter and spawns configured daemons
- Called during `initialize()` after tools setup
- `subagent_node` in `nodes.py` already handles `spawn_cli`/`interact_cli`/`terminate_cli` via `agent._cli_adapter`

### `model_name` Passthrough (P2)

## Notes

- The `ExternalCliAdapter` uses line-delimited JSON protocol. Real CLI tools (like Claude Code) may require adapter-specific message framing — the current implementation is a clean abstraction that can be extended per-tool.
- The `planning_model` wiring uses the same `llm_provider` but different `chat_model`. If the provider does not support the specified model, it falls back gracefully.
- `STEP_COMPLETE` events are emitted by comparing `_prev_step_index` to `plan.current_step_index` in `astream_progress()`. This detects completed steps even when multiple steps advance in a single graph iteration.
