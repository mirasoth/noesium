## Objective

Decouple `noesium.core` from NoeAgent-specific logic (TUI config, console log
policy) so core remains a general agentic framework. Validate end-to-end BU
subagent progress event exposure in TUI.

## Completed

- **Core LoggingConfig generalized**: Restored `level` field (default INFO) for
  both console and file. Added optional `file_level` for file-only override.
  Removed NoeAgent-specific "console always WARNING" from core docstrings.
- **TUIConfig removed from core**: Deleted `TUIConfig` class and `tui` field
  from `NoeAgentConfig`. TUI config now handled in `NoeConfig.from_global_config()`
  via `extra="allow"` dict access.
- **NoeAgent console policy preserved**: `NoeAgent._setup_logging()` and
  `tui.main()` still pass `console_level="WARNING"` — this is application policy,
  not framework default.
- **Environment variable overrides updated**: `LOG_LEVEL` → `level`,
  `NOE_FILE_LOG_LEVEL` → `file_level`. Both work independently.
- **Design doc updated**: Added "Module Layering" section to
  `docs/impl/noeagent-config-system.md` documenting the isolation rules.
  Rewrote logging section to describe two-layer architecture.
- **BU progress pipeline validated**: End-to-end test confirmed 13 subagent
  events (start + 11 progress + end) flow from BrowserUseAgent through
  BuiltInAgentCapabilityProvider → NoeAgent.execute_builtin_subagent_streaming →
  astream_progress → TUI renderer.
- **Cosmetic fix**: Fixed `_describe_action_from_output()` to not display raw
  Python `list.index` method object in BU action descriptions.
- **example_config.json updated**: Logging section now shows `level` + `file_level`.

## Notes

- BU progress events arrive batched (after node completion) rather than real-time
  during execution — this is inherent to the LangGraph streaming model where
  node execution is atomic. Session log callbacks fire immediately though.
- The `NoeAgentConfig` class name remains as-is in core since the config file
  format IS for noeagent. The class itself only contains framework-generic
  sections now.
