# NoeAgent TUI Feature Gaps and Performance Optimizations Implementation

## Objective

Implement all feature gaps identified in `docs/impl/noeagent-tui-analysis.md` and all performance optimizations proposed in `docs/impl/noeagent-perf-opt.md`.

## Completed

### Performance Optimizations

- **O1 (Init Guard + Cached Graph)**: Added `_initialized` flag to `initialize()` so setup runs only once per agent lifetime. Added `_get_compiled_graph()` that caches the compiled LangGraph graph and invalidates only on mode change. Added `reinitialize()` for explicit re-init after config changes.
- **O2 (Shared Toolkit Instances)**: `spawn_subagent()` now shares parent's `_registry`, `_tool_executor`, and creates a child-specific `ToolContext` (with scoped `agent_id`). Eliminates ~200-500ms toolkit re-loading per subagent.
- **O3 (Parallel Subagent Execution)**: Added `interact_with_subagents()` method that uses `asyncio.gather()` for concurrent subagent execution when multiple independent interactions are needed.
- **O4 (Cached Tool Descriptions)**: Added `get_tool_descriptions()` on NoeAgent that caches the formatted tool description string and invalidates when registry provider count changes. `execute_step_node` accepts `tool_desc_cache` parameter.
- **O5 (Lazy Planning LLM)**: Replaced eager LLM client creation in `initialize()` with `_get_planning_llm()` property that creates on first access.
- **O6 (Event Store Trimming)**: Added trimming check at the start of `astream_progress()` â€” if `InMemoryEventStore` exceeds 10,000 events, trims to last 5,000.

### Feature Gaps

- **Subagent Progress Propagation**: Rewrote `interact_with_subagent()` to iterate `child.astream_progress()` instead of `child.arun()`. Each child event is wrapped as `SUBAGENT_PROGRESS` with `child_event_type` preserved in metadata. `SUBAGENT_END` is emitted when child completes.
- **Callback-to-Yield Bridge**: Added `_subagent_event_queue` (asyncio.Queue) that collects subagent events fired inside nodes. The `astream_progress()` generator drains this queue between LangGraph iterations and in the finally block.
- **SubagentTracker**: New TUI class tracking per-subagent state (plan steps, completed count, current step, status). Renders compact progress lines: `[sub-1] > 2/3 Â· Searching...` or `[sub-1] âś“ 3/3 done`. Max 3 concurrent tracks displayed.
- **TUI SUBAGENT_PROGRESS/END Handling**: Updated `_process_query()` to feed all subagent events into `SubagentTracker`, add step details for subagent plan creation and tool use, and show completion markers.
- **Mode Switch Invalidation**: `/mode` slash command now resets `_initialized`, `_compiled_graph`, `_compiled_mode`, and `_tool_desc_cache` so next query re-initializes for the new mode.

## Notes

- The `_subagent_event_queue` approach ensures subagent events appear in both `astream_progress()` yields (for TUI) and `progress_callbacks` (for library consumers), maintaining the consistency contract.
- Shared toolkit infrastructure (O2) requires careful isolation: only `_registry` and `_tool_executor` are shared; the `_tool_context` is child-specific; memory providers are not shared.
- The O6 trimming is conservative (10K threshold, keep 5K) and only applies to `InMemoryEventStore` which is used in dev/TUI; production `FileEventStore` has its own append-only persistence.
