## Objective

Implement all remaining gaps in RFC-1005 (Autonomous Architecture), RFC-1006 (Goal Engine), and RFC-1007 (Event System) to achieve 100% spec compliance.

## Completed

### Phase 1: RFC-1005 Completion (CognitiveLoop)

1. **Subagent Integration** - Complete subagent invocation via SubagentManager
   - Added `_subagent_manager` parameter to CognitiveLoop
   - Added `_session_id` for context tracking
   - Implemented full SubagentManager.invoke_stream() integration in `_execute_decision()`

2. **Pause/Resume** - Added pause/resume capability
   - Added `_paused` flag and `_pause_event` asyncio.Event
   - Implemented `pause()`, `resume()`, `is_paused` property
   - Loop waits on pause_event in run() before each tick

3. **Tick Metrics** - Added CognitiveLoopMetrics dataclass
   - Tracks: total_ticks, successful_ticks, failed_ticks, idle_ticks
   - Tracks: total_tick_duration_ms, last_tick_duration_ms
   - Added avg_tick_duration_ms property and reset() method

### Phase 2: RFC-1006 Completion (GoalEngine)

1. **Goal Dependencies** - Added blocked_by field to Goal model
   - Goals with incomplete dependencies are skipped in next_goal()
   - Added `add_dependency()` and `remove_dependency()` methods

2. **Goal Timeout/Deadline** - Added deadline field to Goal model
   - Added `check_timeouts()` method that auto-fails expired goals
   - Timeout failures don't trigger retry (allow_retry=False)

3. **Goal Retry Policy** - Added retry_count and max_retries to Goal model
   - fail_goal() now retries if retries remain (status -> PENDING)
   - Configurable max_retries (default: 3)
   - allow_retry parameter to force permanent failure

### Phase 3: RFC-1007 Completion (Event System)

1. **Watchdog FileSystemEventSource** - Native file system events
   - Added WatchdogFileSystemEventSource using watchdog library
   - Added _WatchdogEventHandler for create/delete/modify/move events
   - Added get_filesystem_event_source() factory function
   - Added WATCHDOG_AVAILABLE flag for feature detection
   - watchdog is optional dependency (pip install noeagent[autonomous])

2. **Event Deduplication** - Added to EventQueue
   - Events hashed by type, source, and payload
   - Configurable dedup_window_seconds (default: 5.0)
   - enqueue() returns bool (True if enqueued, False if duplicate)
   - Added dedup_cache_size property

3. **Event Replay** - New EventReplayer class
   - replay() async iterator with filters (event_type, source, since, until, limit)
   - replay_to_queue() for reprocessing events
   - count_events(), get_event_types(), get_event_sources() helpers

### Phase 4: Integration Updates

1. **AutonomousRunner** - Updated to use all new features
   - Passes SubagentManager to CognitiveLoop
   - Added watch_path and prefer_watchdog parameters
   - Uses get_filesystem_event_source() for file watching
   - Added pause(), resume(), is_paused, metrics helpers

### Phase 5: Exports and Tests

1. **Updated __init__.py** - All new classes exported
2. **Added pyproject.toml** - watchdog optional dependency
3. **Created test_autonomous_complete.py** - Comprehensive tests

## Notes

Key implementation decisions:
- Timeout failures don't retry (allow_retry=False) to prevent infinite timeout loops
- Watchdog is optional - falls back to polling if not installed
- Dedup window is configurable (default 5s) for different use cases
- EventReplayer supports both iteration and queue injection
