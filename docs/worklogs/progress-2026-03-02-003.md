## Notes

- The `AgentAction.exactly_one_action` validator is lenient: if no action is set, it falls back to `text_response = thought` rather than raising. This prevents validation errors from breaking the agent loop.
- The `_coerce_tool_args` function handles common LLM mistakes (string instead of array, string instead of int/float/bool).
- CLI subagent adapter (`_cli_adapter`) is checked via `getattr` — the actual `ExternalCliAdapter` implementation is deferred to a future session.
