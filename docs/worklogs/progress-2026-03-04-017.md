# Built-in subagent factory binding fix

## Objective
Fix the bug where explicit numeric prefix selection (`2 ...`) intended for `browser_use` could instantiate and run `tacitus` instead.

## Completed
- Fixed built-in subagent registration to bind each provider factory to the correct per-loop factory callable.
- Added a regression test ensuring `browser_use` and `tacitus` providers each invoke their own creation method.
- Attempted to run targeted integration tests for the fix.

## Files Changed
- `noesium/noeagent/agent.py` - fixed closure binding in `_setup_builtin_subagents` so provider factories no longer capture the last loop value.
- `tests/integration/test_noe_subagent_integration.py` - added regression test for correct factory binding per built-in subagent.

## Tests
- `uv run pytest tests/integration/test_noe_subagent_integration.py -k "binds_correct_factory_per_subagent or registers_providers"` → failed (network/dependency resolution during uv sync).
- `uv run --no-sync pytest tests/integration/test_noe_subagent_integration.py -k "binds_correct_factory_per_subagent or registers_providers"` → failed (import error: `noesium` module not found in current test runtime).
- `PYTHONPATH=. uv run --no-sync pytest tests/integration/test_noe_subagent_integration.py -k "binds_correct_factory_per_subagent or registers_providers"` → failed (missing runtime dependency: `dotenv`).

## Notes
The root cause is Python late binding in a loop closure: the nested `factory()` captured `factory_callable` by reference, so all registered providers used the final loop callable (Tacitus in the reported configuration).
