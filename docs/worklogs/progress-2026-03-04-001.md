## Objective

Optimize NoeAgent logging and session management:
1. Session-isolated logging at `~/.noeagent/sessions/<sessionId>/`
2. Console/TUI logging fixed at WARNING for UX simplicity
3. File-based session logging at configurable INFO level (global)
4. Browser-use temp data in session-isolated directories

## Completed

- Deep analysis of noeagent logging, session management, and config system
- Deep analysis of browser_use module: identified 10+ temp data output locations
- Deep analysis of other subagents/toolkits: tacitus/askura have no temp data
- Updated `setup_logging()` with `console_level` parameter for separate console/file levels
- Updated `LoggingConfig` in core/config.py: `level`/`file` → `file_level` (console is always WARNING)
- Added session_id / session_dir / file_log_level to NoeConfig with auto-derivation
- Updated NoeAgent._setup_logging for session-isolated file logging
- Updated TUI main() to pre-configure logging before NoeAgent creation
- Updated SessionLogger to write `session.jsonl` in session directory
- Updated BrowserUseAgent to accept `parent_session_dir` for session isolation
- Updated NoeAgent._create_browser_use_agent to pass session_dir
- Added backward compat: _normalize_config_data migrates old `logging.level` → `file_level`
- Updated example_config.json with new logging schema
- Updated design doc (noeagent-config-system.md) with new rules
- Fixed all test failures (config test assertions for new field names)

## Notes

- Tacitus and Askura subagents produce no temp files, so no changes needed.
- Toolkit temp data (python_executor, bash, memory, etc.) uses `working_directory` which is the user's project dir — this is intentional and not session-specific.
- The BU Agent (service.py) `agent_directory` in `/tmp/` remains unchanged as it's truly ephemeral. The important persistent data (browser profile, downloads) is now under the noeagent session dir.
- `LOG_LEVEL` env var is kept as backward-compat alias for `NOE_FILE_LOG_LEVEL`.
