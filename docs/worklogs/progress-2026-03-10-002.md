# Notes

## Architecture Decision

The `NoeToolHelper` class extends the base `ToolHelper` from `noesium.utils.tool_utils` rather than replacing it entirely. This approach:

1. **Reuses core infrastructure** - ToolExecutor, ToolContext, CapabilityRegistry creation
2. **Adds NoeAgent features** - Session directories, concurrent loading
3. **Maintains backward compatibility** - `setup_tools()` API unchanged
4. **Follows DRY principle** - Common tool setup logic shared across agents

## Key Differences from Base ToolHelper

| Feature | ToolHelper | NoeToolHelper |
|---------|-----------|---------------|
| Toolkit loading | Sequential | Concurrent (asyncio.gather) |
| Session directories | Not supported | Per-toolkit session dir overrides |
| MCP integration | Not included | Supported via `_setup_mcp_servers` |
