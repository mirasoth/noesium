## Objective

Fix the TUI not displaying real-time progress details from the tacitus subagent while it runs. The spinner showed "Processing..." but no query generation, web search, or reflection events appeared during execution.

## Root Cause

`astream_progress()` in `agent.py` used a simple `asyncio.Queue` drain pattern:

```python
async for raw_event in compiled.astream(initial):
    # drain queue here (BETWEEN nodes only)
    while not queue.empty():
        yield queue.get_nowait()
    # process raw_event...
```

The queue was only drained **between** LangGraph node executions. When `subagent_node` ran and called `execute_builtin_subagent_streaming()`, tacitus events were put into the queue via `put_nowait()` — but `astream_progress()` was blocked waiting for `compiled.astream()` to yield the next event. That only happens after the node finishes. So all tacitus events piled up and were released only after tacitus completed.

## Fix

Replaced the direct `compiled.astream()` iteration with a concurrent background task pattern using a merged queue:

1. `_run_graph` task: runs LangGraph in background, forwards `("graph", raw_event)` to `_merged_q`
2. `_forward_subagent_events` task: reads from `_subagent_event_queue`, forwards `("subagent", event)` to `_merged_q`
3. `astream_progress` reads from `_merged_q`, yielding subagent events in real-time while the node executes

No new tests added; existing tests in `tests/integration/test_noe_subagent_integration.py` cover the event propagation logic.

## Notes

The fix relies on Python asyncio's cooperative multitasking: `_forward_subagent_events` and `astream_progress` are both waiting on queues, and they get CPU time between `await` points in `execute_builtin_subagent_streaming`. The `put_nowait` calls in `execute_builtin_subagent_streaming` wake up `_forward_subagent_events`, which puts items into `_merged_q`, which wakes up `astream_progress` to yield the event — all while `subagent_node` is still running.
