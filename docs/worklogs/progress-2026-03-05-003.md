# RFC-1007 Layered Architecture Compliance Review

## Objective
Review noesium code structure per `docs/specs/RFC-1007.md` so that core, toolkits, and subagents do not reference NoeAgent-specific configs, constants, or docs.

## Completed
- **Core**: Replaced NoeAgent-specific names with framework-neutral ones:
  - `noesium/core/consts.py`: `NOE_AGENT_HOME` → `NOESIUM_HOME` only (default `~/.noesium`); `NOE_AGENT_*` dirs → `NOESIUM_*`; `NOE_TMP_BASE` → `NOESIUM_TMP_BASE`
  - `noesium/core/config.py`: Docstring and env vars `NOE_*` → `NOESIUM_*`; path default via `NOESIUM_HOME`
  - `noesium/core/llm/*`, `tracing/opik_tracing.py`, `toolify/config.py`, `toolify/README.md`: `NOE_OPIK_TRACING` / `NOE_LLM_*` → `NOESIUM_*`
  - Docstrings: removed "NoeAgent" from `core/capability/providers.py`, `core/event/progress.py`, `core/agent/subagent/protocol.py`, `core/agent/subagent/__init__.py`, `core/utils/logging.py`
- **Toolkits**: `bash_toolkit.py` custom prompt default changed from `NOE_BASH_PROMPT>> ` to config-driven default `noesium-bash>> `
- **Subagents**: `subagents/bu/agent/__init__.py` uses `NOESIUM_HOME` from core for fallback session dir; docstrings say "caller" / "application" instead of NoeAgent. `subagents/tacitus/agent.py` docstring updated to "core progress system"
- **NoeAgent layer**: `get_noe_config_path()` added; `from_global_config()` and CLI use it and apply `NOE_*` overrides for backward compat. Progress imports switched from removed `noeagent.progress` to `noesium.core.event`
- **Top-level**: `NoeAgentConfig` alias kept in `noesium/__init__.py` (not in core). Tests updated to use `FrameworkConfig` and `NOESIUM_*` where testing core; noeagent CLI tests still use `NOE_AGENT_CONFIG` for app path
