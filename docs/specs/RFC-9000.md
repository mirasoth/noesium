## 1. Abstract

Voyager is a personal coding assistant webserver built on the NoeAgent framework. It provides a web-based interface for autonomous code generation, analysis, and development tasks. Unlike multi-user collaborative platforms, Voyager targets the individual developer workflow—offering a streamlined, single-user experience without authentication complexity. This RFC defines the design philosophy, core principles, and conceptual model that guide Voyager's development.

## 3. Background & Motivation

Modern AI coding assistants fall into two categories:

1. **IDE-embedded assistants** (GitHub Copilot, Cursor): Tight editor integration but limited autonomy
2. **Web-based platforms** (ChatGPT Codex, Replit): Full autonomy but multi-user complexity

Voyager fills the gap: a **personal**, **autonomous** coding assistant with a clean web interface. It leverages NoeAgent's planning, execution, and reflection capabilities while providing a focused, single-user experience.

**Key Insight**: Most developers don't need multi-user collaboration for AI-assisted coding. They need a reliable, personal assistant that understands their codebase and can work autonomously.

## 4. Design Philosophy

### 4.1 Personal Over Collaborative

Voyager is designed for the **individual developer**. This eliminates:

* Authentication and authorization complexity
* Multi-tenant data isolation concerns
* Real-time collaboration infrastructure
* Permission management overhead

Instead, Voyager focuses on:

* Deep codebase understanding
* Persistent context and memory
* Fast, responsive interactions
* Reliable autonomous execution

### 4.2 Task-Centric Workflow

Voyager organizes work around **tasks**—not chats, not files, not branches. A task represents a discrete unit of work:

* "Refactor the authentication module"
* "Add unit tests for the payment handler"
* "Migrate the database schema"

Tasks are first-class entities with:
* Clear lifecycle (created → in_progress → completed)
* Associated code changes and artifacts
* Execution history and reasoning traces
* Persistent memory for future reference

### 4.3 Autonomous Execution

Voyager doesn't just suggest code—it **executes** tasks autonomously:

* **Planning**: Decomposes goals into actionable steps
* **Execution**: Reads, writes, and modifies code
* **Reflection**: Evaluates results and adapts approach
* **Integration**: Commits changes to GitHub repositories

The user initiates a task, then reviews the results—not the process.

### 4.4 Transparent Reasoning

While Voyager operates autonomously, it maintains **full transparency**:

* Every decision is logged with reasoning
* Code changes are traced to specific steps
* Tool invocations are visible in real-time
* Memory and context are inspectable

Users can understand *why* Voyager made each decision.

## 5. Core Principles

### 5.1 Single User, Single Context

Voyager serves one user. All state, memory, and context belong to that user. This enables:

* Simplified architecture without tenant isolation
* Deep personalization and learning
* Persistent long-term memory
* Trust through transparency

**Invariant**: No authentication layer; the system assumes trusted local access.

### 5.2 Task as Primary Abstraction

Tasks are the central organizing unit. Everything revolves around tasks:

* **Navigation**: Browse tasks, not chats or files
* **History**: Review past tasks and their outcomes
* **Memory**: Context is task-scoped with cross-task retrieval
* **Artifacts**: Code changes, documents, and results are task-linked

**Invariant**: Every user action creates or references a task.

### 5.3 Code-First, Not Chat-First

Voyager is not a chatbot. It's a **coding agent**:

* Primary output is code, not text
* Changes are applied directly to repositories
* Git operations are first-class actions
* Code analysis drives decision-making

**Invariant**: Text responses support code; code is never secondary.

### 5.4 GitHub as Source of Truth

Voyager operates on real codebases:

* Repositories are cloned and managed locally
* Changes are committed back to GitHub
* Branches and PRs are native concepts
* Version history is preserved

**Invariant**: Voyager never owns code; it operates on GitHub repositories.

### 5.5 Streaming Progress, Not Final Answers

Long-running tasks require **progress visibility**:

* Real-time step completion indicators
* Tool invocation streaming
* Intermediate results display
* Thinking and reasoning visualization

**Invariant**: Users see progress in real-time, not just final results.

## 6. Conceptual Model

### 6.1 Entity Model

```
User (implicit)
  │
  ├── Repository ──────┬── Branch
  │                     │
  │                     └── Commit
  │
  └── Task ─────────────┬── Step
                        │
                        ├── Message
                        │
                        ├── CodeChange
                        │
                        └── Artifact
```

### 6.2 Task Lifecycle

```
created ──▶ planning ──▶ executing ──▶ reflecting ──▶ completed
    │            │            │              │            │
    └────────────┴────────────┴──────────────┴────────────┘
                           may loop
```

### 6.3 Navigation Model

The UI provides two primary views:

1. **Code**: Repository browser, file viewer, code editor
2. **Docs**: Documentation, RFCs, knowledge base

Both views center on **Tasks**:

* Task list shows all tasks (active, completed)
* Task detail shows execution timeline, changes, artifacts

## 7. Integration Boundaries

### 7.1 NoeAgent Integration

Voyager wraps NoeAgent as its execution engine:

* **Planning**: Uses NoeAgent's `TaskPlanner` for goal decomposition
* **Execution**: Uses NoeAgent's tool system for code operations
* **Memory**: Uses NoeAgent's memory providers for context
* **Streaming**: Uses NoeAgent's `astream_progress()` for real-time updates

**Constraint**: Voyager does not modify NoeAgent's core logic; it provides a web interface.

### 7.2 GitHub Integration

GitHub integration provides:

* **Clone**: Fetch repositories to local workspace
* **Pull**: Sync with remote branches
* **Commit**: Apply changes and commit
* **Push**: Push commits to remote
* **Branch**: Create and manage branches

**Constraint**: Voyager operates on cloned repositories; it doesn't use GitHub API for file operations.

### 7.3 Web Interface

The web interface provides:

* **REST API**: HTTP endpoints for task management
* **WebSocket**: Real-time progress streaming
* **Static Frontend**: Single-page application for UI

**Constraint**: Frontend is lightweight; no build complexity, minimal dependencies.

## 8. Design Constraints

### 8.1 Simplicity Over Features

Voyager prioritizes simplicity:

* No authentication system
* No user management
* No real-time collaboration
* No complex configuration

Features are added only when they serve the core use case: **autonomous coding assistance**.

### 8.2 Local-First Deployment

Voyager runs locally:

* No cloud infrastructure required
* Data stored in local filesystem
* Repositories cloned to local workspace
* Memory persisted in local files

This ensures:
* Privacy: Code never leaves the developer's machine
* Performance: Low latency, no network overhead
* Reliability: No external dependencies

## 9. User Experience Vision

### 9.1 Onboarding

1. User starts Voyager server
2. UI opens to task list (empty initially)
3. User connects a GitHub repository
4. User creates first task via natural language input
5. Voyager executes and displays progress
6. User reviews results and creates follow-up tasks

### 9.2 Daily Workflow

1. **Morning**: Review overnight task completions
2. **Development**: Create tasks for complex refactoring
3. **Review**: Inspect code changes and reasoning
4. **Iteration**: Provide feedback through new tasks

### 9.3 Task Creation

User input is natural language:

```
"Add error handling to the user registration flow"
```

Voyager:
1. Parses the request
2. Identifies relevant code (registration module)
3. Plans the changes
4. Executes modifications
5. Commits to a feature branch
6. Presents summary for review

## 12. Conclusion

Voyager embodies a focused vision: a personal, autonomous coding assistant with a clean, task-centric web interface. By eliminating multi-user complexity and prioritizing code operations, Voyager delivers a streamlined experience for developers who want AI assistance without collaboration overhead.

> Voyager is not a chatbot that writes code; it's an agent that executes coding tasks.
