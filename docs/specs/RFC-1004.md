## 1. Abstract

This RFC defines the implementation architecture for the unified capability registry and discovery system in Noesium. It implements the design from RFC-0005 (Capability Registry and Discovery Protocol) using a **Provider-First, Hybrid Event-Sourced** approach.

The registry unifies tools (built-in, MCP), skills, and agents (in-process, CLI) under a single `CapabilityProvider` protocol. Providers are the source of truth for fast reads; an append-only changelog supports audit and optional event-store persistence. Deterministic resolution, health-aware routing, and event-mediated invocation enable loose coupling and composable multi-agent workflows.

## 3. Core Types

### 3.1 CapabilityType Enum

```python
class CapabilityType(str, Enum):
    TOOL = "tool"           # Built-in or LangChain tool (InProcessTool)
    MCP_TOOL = "mcp_tool"   # MCP server tool (RemoteTool)
    SKILL = "skill"         # Composed tool workflow
    AGENT = "agent"         # In-process subagent (Subagent)
    CLI_AGENT = "cli_agent" # External CLI subagent (ExternalCliSubagent)
```

**Stateless vs Stateful:**

```python
STATEFUL_TYPES = frozenset({CapabilityType.AGENT, CapabilityType.CLI_AGENT})
```

- **Stateless** (`TOOL`, `MCP_TOOL`, `SKILL`): No persistent state; `health()` always returns `True`; no background monitoring.
- **Stateful** (`AGENT`, `CLI_AGENT`): Session-scoped state; require periodic health polling; cached health used by `resolve()`.

### 3.2 Classification Enums

```python
class DeterminismClass(str, Enum):
    DETERMINISTIC = "deterministic"
    NONDETERMINISTIC = "nondeterministic"

class SideEffectClass(str, Enum):
    PURE = "pure"
    IDEMPOTENT = "idempotent"
    EXTERNAL = "external"

class LatencyClass(str, Enum):
    FAST = "fast"     # < 100ms
    MEDIUM = "medium" # < 5s
    SLOW = "slow"     # < 60s
    LONG = "long"     # > 60s
```

### 3.3 CapabilityDescriptor

Pydantic model describing a capability contract:

```python
class CapabilityDescriptor(BaseModel):
    capability_id: str
    version: str = "1.0.0"
    capability_type: CapabilityType
    description: str = ""
    input_schema: dict[str, Any] = Field(default_factory=dict)
    output_schema: dict[str, Any] = Field(default_factory=dict)
    determinism: DeterminismClass = DeterminismClass.NONDETERMINISTIC
    side_effects: SideEffectClass = SideEffectClass.EXTERNAL
    latency: LatencyClass = LatencyClass.FAST
    tags: list[str] = Field(default_factory=list)

    @property
    def stateful(self) -> bool:
        """Derived from capability_type; True for AGENT and CLI_AGENT."""
        return self.capability_type in STATEFUL_TYPES
```

### 3.4 CapabilityProvider Protocol

```python
@runtime_checkable
class CapabilityProvider(Protocol):
    """Anything that provides a capability: tool, MCP tool, skill, or agent."""

    @property
    def descriptor(self) -> CapabilityDescriptor: ...

    async def invoke(self, **kwargs: Any) -> Any: ...

    async def health(self) -> bool: ...
```

- **descriptor**: Immutable metadata; used for discovery and resolution.
- **invoke**: Async execution; semantics depend on provider type (see §4).
- **health**: Liveness check; stateless providers return `True`; stateful providers perform actual checks.

### 3.5 CapabilityQuery

Structured query for discovery:

```python
class CapabilityQuery(BaseModel):
    capability_id: str | None = None
    version: str | None = None
    capability_type: CapabilityType | None = None
    tag: str | None = None
    determinism: DeterminismClass | None = None
    healthy_only: bool = True
```

## 4. Provider Adapters

Each adapter wraps an existing Noesium construct into the `CapabilityProvider` protocol.

### 4.1 ToolCapabilityProvider

**Purpose**: Wraps a built-in or LangChain `AtomicTool` as a stateless capability.

**Constructor**:
```python
def __init__(
    self,
    tool: AtomicTool,
    executor: ToolExecutor,
    context: ToolContext,
) -> None
```

**Invoke semantics**: Calls `executor.run(tool, context, **kwargs)`. The `ToolExecutor` emits `tool.invoked` and `tool.completed` (or `tool.failed`) events.

**Health semantics**: Always returns `True` (stateless).

**Descriptor**: `capability_id` = `tool.name`; `capability_type` = `TOOL`; determinism/side_effects mapped from tool metadata.

---

### 4.2 MCPCapabilityProvider

**Purpose**: Wraps an MCP-sourced `AtomicTool` as a stateless remote capability.

**Constructor**:
```python
def __init__(
    self,
    tool: AtomicTool,
    executor: ToolExecutor,
    context: ToolContext,
) -> None
```

**Invoke semantics**: Same as `ToolCapabilityProvider` — `executor.run(tool, context, **kwargs)`. MCP tools are inherently async; the executor handles the call.

**Health semantics**: Always returns `True` (stateless). MCP session liveness is not modeled at the capability level.

**Descriptor**: `capability_id` = `tool.name`; `capability_type` = `MCP_TOOL`; typically `determinism=EXTERNAL`, `side_effects=EXTERNAL`.

---

### 4.3 SkillCapabilityProvider

**Purpose**: Wraps a `Skill` (composed tool workflow) as a capability.

**Constructor**:
```python
def __init__(
    self,
    skill: Skill,
    tool_registry: ToolRegistry,
    tool_executor: ToolExecutor,
    context: ToolContext,
) -> None
```

**Invoke semantics**: Calls `skill.execute(tool_registry=..., tool_executor=..., context=..., **kwargs)`. The skill orchestrates multiple tool calls internally.

**Health semantics**: Always returns `True` (stateless).

**Descriptor**: `capability_id` = `skill.name`; `capability_type` = `SKILL`; input/output schema from skill definition.

---

### 4.4 AgentCapabilityProvider

**Purpose**: Wraps an in-process `NoeAgent` (or other `BaseGraphicAgent` child) as a stateful capability.

**Constructor**:
```python
def __init__(
    self,
    name: str,
    agent: BaseGraphicAgent,
    *,
    task_types: list[str] | None = None,
) -> None
```

**Invoke semantics**: Extracts `message` or `task` from `kwargs` and calls `agent.arun(message)`. The agent runs its LangGraph workflow internally.

**Health semantics**: Returns `agent is not None`. The background health monitor polls this; if the agent reference is cleared, the capability is marked unhealthy.

**Descriptor**: `capability_id` = `f"agent:{name}"`; `capability_type` = `AGENT`; `stateful=True`; tags include `subagent`, `agent:{name}`, and optional `task_types`.

---

### 4.5 CliAgentCapabilityProvider

**Purpose**: Wraps an `ExternalCliAdapter` handle as a stateful CLI subagent capability.

**Constructor**:
```python
def __init__(
    self,
    name: str,
    adapter: ExternalCliAdapter,
    *,
    task_types: list[str] | None = None,
) -> None
```

**Invoke semantics**: Extracts `message` or `task` from `kwargs` and calls `adapter.interact(name, message)`. The adapter manages stdin/stdout communication with the external CLI process.

**Health semantics**: Delegates to `adapter.health_check(name)`. The adapter performs process-level liveness checks (e.g., process alive, pipe responsive).

**Descriptor**: `capability_id` = `f"cli_agent:{name}"`; `capability_type` = `CLI_AGENT`; `stateful=True`; tags include `cli_subagent`, `cli:{name}`.

## 5. CapabilityRegistry

### 5.1 Constructor

```python
def __init__(
    self,
    event_store: EventStore | None = None,
    producer: AgentRef | None = None,
    health_interval: float = 10.0,
) -> None
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `event_store` | `EventStore \| None` | Optional; when set, `CapabilityRegistered` events are flushed asynchronously for persistence and replay. |
| `producer` | `AgentRef \| None` | Agent identity for emitted events; defaults to `agent_id="registry"`, `agent_type="system"`. |
| `health_interval` | `float` | Seconds between health polls for stateful providers; default 10.0. |

### 5.2 Internal State

| Field | Type | Purpose |
|-------|------|---------|
| `_providers` | `dict[str, list[CapabilityProvider]]` | capability_id → list of providers (supports multiple versions). |
| `_by_name` | `dict[str, CapabilityProvider]` | capability_id → last-registered provider; O(1) lookup. |
| `_changelog` | `list[RegistryEvent]` | Append-only audit log of all mutations. |
| `_listeners` | `list[RegistryListener]` | Observers notified on registration, unregistration, health change. |
| `_health_cache` | `dict[str, bool]` | capability_id → cached health; only for stateful providers. |
| `_event_store` | `EventStore \| None` | Optional event store for hybrid ES. |
| `_producer` | `AgentRef` | Producer for emitted events. |
| `_health_task` | `asyncio.Task \| None` | Background health monitor task. |

### 5.3 RegistryEvent

Lightweight audit record emitted on every registry mutation:

```python
@dataclass
class RegistryEvent:
    action: str  # "registered" | "unregistered" | "health_changed"
    capability_id: str
    version: str
    capability_type: str
    timestamp: float = field(default_factory=time.time)
    detail: dict[str, Any] = field(default_factory=dict)
```

### 5.4 Registration

**register(provider: CapabilityProvider) -> None**

- Appends provider to `_providers[capability_id]`.
- Updates `_by_name[capability_id]` to the new provider.
- If stateful, initializes `_health_cache[capability_id] = True`.
- Appends `RegistryEvent(action="registered", ...)` to `_changelog`.
- Notifies listeners.
- Calls `_flush_event_async(descriptor)` to persist to EventStore (fire-and-forget).

**register_many(providers: list[CapabilityProvider]) -> None**

- Calls `register(p)` for each provider.

**unregister(capability_id: str, version: str | None = None) -> None**

- Removes providers matching `capability_id` (and optionally `version`).
- Updates `_providers`, `_by_name`, `_health_cache`.
- Appends `RegistryEvent(action="unregistered", ...)` to `_changelog`.
- Notifies listeners.

### 5.5 Discovery

**find(capability_id: str, version: str | None = None) -> list[CapabilityProvider]**

- Returns providers matching `capability_id`; optional `version` prefix filter.

**find_by_type(cap_type: CapabilityType) -> list[CapabilityProvider]**

- Returns all providers of the given type.

**find_by_tag(tag: str) -> list[CapabilityProvider]**

- Returns providers whose descriptor has `tag` in `tags`.

**find_by_determinism(cls: DeterminismClass) -> list[CapabilityProvider]**

- Returns providers matching the determinism class.

**query(q: CapabilityQuery) -> list[CapabilityProvider]**

- Structured query across all filters; when `healthy_only=True`, excludes stateful providers with cached `health=False`.

### 5.6 Resolution

**async resolve(capability_id: str, version: str | None = None) -> CapabilityProvider**

- Deterministic resolution: first healthy match by registration order.
- For stateless providers, health is always considered True.
- For stateful providers, uses `_health_cache` (updated by background task).
- Raises `CapabilityNotFoundError` if no healthy match.

**get_by_name(name: str) -> CapabilityProvider**

- O(1) lookup by `capability_id`; returns last-registered provider.
- Raises `CapabilityNotFoundError` if not found.

### 5.7 Observer

**add_listener(listener: RegistryListener) -> None**

- Appends a callable `(RegistryEvent) -> Any` to `_listeners`.
- Listeners are invoked synchronously on each mutation; exceptions are logged and swallowed.

### 5.8 Health Monitoring

**start_health_monitor() -> None**

- Starts background `asyncio` task that runs `_health_loop()`.

**stop_health_monitor() -> None**

- Cancels the health task; awaits cancellation.

**_health_loop() -> None**

- Infinite loop: `await asyncio.sleep(health_interval)` then `await _poll_stateful_health()`.
- Catches `asyncio.CancelledError` to exit cleanly.

**_poll_stateful_health() -> None**

- For each stateful provider, calls `await provider.health()`.
- Updates `_health_cache[capability_id]`.
- If health changed, appends `RegistryEvent(action="health_changed", ...)` and notifies listeners.

**get_health(capability_id: str) -> bool | None**

- Returns cached health; `None` if not tracked (e.g., stateless).

### 5.9 Hybrid Event Sourcing

**_flush_event_async(descriptor: CapabilityDescriptor) -> None**

- Fire-and-forget: schedules `_flush_event(descriptor)` via `loop.create_task()`.
- No-op if `_event_store` is None or no running event loop.

**async _flush_event(descriptor: CapabilityDescriptor) -> None**

- Constructs `CapabilityRegistered` domain event with full descriptor payload.
- Converts to `EventEnvelope` and appends to `_event_store`.
- Exceptions are logged; no propagation.

**async rebuild_from_events(event_store: EventStore) -> int**

- Reads events from store; replays `capability.registered` and `capability.deprecated` into `_changelog`.
- Does **not** reconstruct live providers — those must be re-registered by the agent on startup.
- Returns count of replayed events.
- Use case: cold-start audit replay, projection rebuild.

## 6. Invocation Flow

### 6.1 Tools (TOOL, MCP_TOOL, SKILL)

**Direct invocation** — no capability-level events:

1. Caller obtains provider: `provider = registry.resolve(name)` or `registry.get_by_name(name)`.
2. Caller invokes: `result = await provider.invoke(**args)`.
3. For tools, `ToolExecutor.run()` emits `tool.invoked` and `tool.completed` (or `tool.failed`) internally.

No `capability.requested` / `capability.invoked` / `capability.completed` at the capability layer for tools; tool events suffice for observability.

### 6.2 Agents (AGENT, CLI_AGENT)

**Event-mediated invocation** (RFC-0005):

1. Caller emits `capability.requested` (or equivalent task delegation event).
2. Kernel/orchestrator resolves: `provider = await registry.resolve(capability_id)`.
3. Emit `capability.invoked` (caller, target, capability_id, correlation_id).
4. Execute: `result = await provider.invoke(message=...)`.
5. Emit `capability.completed` (result, error).

Subagent invocation may involve session creation, task queuing, and lifecycle transitions per RFC-0005 §7.2 and §10–11.

## 7. Health Model

### 7.1 Stateless Providers (TOOL, MCP_TOOL, SKILL)

- `health()` always returns `True`.
- No background monitoring.
- `resolve()` and `query(healthy_only=True)` always include them.

### 7.2 Stateful Providers (AGENT, CLI_AGENT)

- `health()` performs actual liveness check:
  - `AgentCapabilityProvider`: `agent is not None`.
  - `CliAgentCapabilityProvider`: `await adapter.health_check(name)`.
- Background asyncio task polls `health()` at `health_interval` seconds.
- Cached status in `_health_cache`; used by `resolve()` and `query(healthy_only=True)`.
- Health transitions emit `RegistryEvent(action="health_changed", ...)` and notify listeners.

## 8. BaseAgent Capability Interface

### 8.1 Declaration Method

`BaseAgent` gains an optional method:

```python
def declare_capabilities(self) -> list[CapabilityDescriptor]:
    """Return capabilities this agent provides. Default: empty list."""
    return []
```

Concrete agents override to declare their capabilities (e.g., subagents, skills).

### 8.2 Registration by NoeAgent

NoeAgent (and similar orchestrators) register capabilities via provider adapters:

- Toolkit tools → `ToolCapabilityProvider` → `registry.register()`
- MCP tools → `MCPCapabilityProvider` → `registry.register()`
- Skills → `SkillCapabilityProvider` → `registry.register()`
- Subagents → `AgentCapabilityProvider` → `registry.register()`
- CLI subagents → `CliAgentCapabilityProvider` → `registry.register()`

## 9. NoeAgent Integration

### 9.1 _setup_tools() Flow

1. Create `CapabilityRegistry(event_store=self._event_store, producer=AgentRef(...), health_interval=10.0)`.
2. Create `ToolExecutor` and `ToolContext`.
3. For each enabled toolkit:
   - Create toolkit via `ToolkitRegistry.create_toolkit()`.
   - Build async toolkits.
   - Convert to `AtomicTool[]` via `BuiltinAdapter.from_toolkit()`.
   - For each tool: `registry.register(ToolCapabilityProvider(tool, executor, context))`.
4. For each MCP server:
   - Connect and discover tools.
   - For each MCP tool: `registry.register(MCPCapabilityProvider(tool, executor, context))`.
5. For custom tools: `registry.register(ToolCapabilityProvider(FunctionAdapter.from_function(func), executor, context))`.
6. If subagents enabled: register `AgentCapabilityProvider` for each subagent.
7. If CLI subagents enabled: register `CliAgentCapabilityProvider` for each.
8. Start health monitor: `registry.start_health_monitor()`.
9. Store `registry` as `self._capability_registry`; use in `tool_node` and `subagent_node`.

### 9.2 tool_node

- Resolve tool by name: `provider = registry.get_by_name(tool_name)` (or `resolve()` for version-aware lookup).
- Invoke: `result = await provider.invoke(**coerced_args)`.
- For subagent tool call: resolve `spawn_subagent` or agent capability; invoke with task.

### 9.3 subagent_node

- Resolve agent capability: `provider = await registry.resolve(f"agent:{name}")` or `registry.get_by_name(f"agent:{name}")`.
- Emit `capability.invoked` (if event-mediated mode).
- Invoke: `result = await provider.invoke(message=task)`.
- Emit `capability.completed`.

## 13. Conclusion

RFC-1004 defines the complete implementation architecture for the Capability Registry. The Provider-First, Hybrid Event-Sourced design ensures:

- **Fast reads**: In-memory provider dict is the source of truth.
- **Audit trail**: Changelog and optional EventStore persistence.
- **Deterministic resolution**: First healthy match by registration order.
- **Health-aware routing**: Stateful providers monitored; unhealthy excluded from resolve.
- **Unified interface**: Tools, MCP tools, skills, and agents all implement `CapabilityProvider`.

> **Every capability is a provider, every provider has a descriptor, and the registry is the single entry point for discovery and resolution.**
