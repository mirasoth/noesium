## 1. Abstract

This RFC defines CognitiveContext, a minimal cognitive state model for maintaining context continuity across TUI conversations and subagent interactions. It provides a lightweight alternative to full event-sourced projections, using a 3-field model (goal, findings, scratchpad) with optional memory integration.

---

## 2. Scope and Non-Goals

### 2.1 Scope

This RFC defines:

* CognitiveContext data model and API
* Integration with NoeAgent's event loop
* Context scoping for subagent delegation
* Optional memory persistence integration

### 2.2 Non-Goals

This RFC does **not** define:

* Event-sourced state reconstruction (see RFC-1002 for projection model)
* Full cognitive architecture (deferred to future RFCs)
* LLM prompt engineering for context injection

---

## 3. Background & Motivation

NoeAgent lacks persistent cognitive state across TUI turns. When a user interacts with the agent over multiple exchanges, context from previous turns is lost. Additionally, when delegating to subagents, there's no structured way to pass relevant context.

**Problems solved by CognitiveContext**:

1. **Context Loss**: Findings from tool calls and planning are forgotten between turns
2. **Subagent Isolation**: Subagents operate without knowledge of parent agent's context
3. **Memory Bloat**: Full event-sourced projections are heavyweight for simple context needs

---

## 4. Design Principles

1. **Minimality**: Three fields capture essential cognitive state (goal, findings, scratchpad)
2. **FIFO Trimming**: Auto-trim findings to prevent unbounded growth
3. **Isolation**: Subagent context is scoped to prevent pollution
4. **Optional Persistence**: Memory integration is opt-in, not required

---

## 5. Data Model

### 5.1 CognitiveContext Fields

```python
class CognitiveContext(BaseModel):
    """Minimal cognitive state for context continuity."""
    
    goal: str = ""
    """Current high-level goal or user intent."""
    
    findings: list[str] = Field(default_factory=list)
    """Recent discoveries from tool calls, observations, or reasoning."""
    
    scratchpad: dict[str, Any] = Field(default_factory=dict)
    """Arbitrary key-value storage for plans, intermediate results, etc."""
    
    max_findings: int = 8
    """Maximum findings to retain (FIFO eviction)."""
```

### 5.2 Field Semantics

| Field | Purpose | Update Trigger | Persistence |
|-------|---------|----------------|-------------|
| `goal` | User intent | Start of conversation turn | Per-session |
| `findings` | Tool/observation results | TOOL_END, SUBAGENT_END events | Auto-trimmed |
| `scratchpad` | Plans, intermediate state | PLAN_CREATED events | Per-session |

## 6. API Specification

### 6.1 Core Methods

```python
def set_goal(self, goal: str) -> None:
    """Set the current goal."""
    
def add_finding(self, finding: str) -> None:
    """Add a finding with FIFO auto-trim."""
    
def set_scratchpad(self, key: str, value: Any) -> None:
    """Store arbitrary data in scratchpad."""
    
def get_scratchpad(self, key: str, default: Any = None) -> Any:
    """Retrieve scratchpad data."""
    
def export(self, include_scratchpad: bool = False) -> str:
    """Export context as markdown for prompt injection."""
    
def for_subagent(self, task: str) -> CognitiveContext:
    """Create scoped context for subagent delegation."""
    
def clear(self) -> None:
    """Reset all context state."""
```

### 6.2 Memory Integration (Optional)

## 7. NoeAgent Integration

### 7.1 Initialization

```python
class NoeAgent:
    def __init__(self, ...):
        self._context = CognitiveContext(
            max_findings=self.config.context_max_findings
        )
```

### 7.2 Event Loop Integration

In `astream_progress()`:

```python
async def astream_progress(self, message: str, ...):
    # Set goal at turn start
    self._context.set_goal(message)
    
    # Optional: enrich from memory
    if self.config.context_auto_recall:
        await self._context.enrich(self.memory_manager)
    
    # Process events
    async for event in self._process_turn(...):
        self._update_context_from_event(event)
        yield event
```

### 7.3 Event-to-Context Mapping

| Event Type | Context Update |
|------------|----------------|
| `TOOL_END` | `add_finding(f"{tool_name}: {result[:80]}")` |
| `SUBAGENT_END` | `add_finding(f"[{subagent_id}] {detail[:80]}")` |
| `PLAN_CREATED` | `set_scratchpad("current_plan", plan_snapshot)` |

## 8. Subagent Context Scoping

### 8.1 Context Delegation

When invoking a subagent:

```python
child_context = self._context.for_subagent(task=message)
subagent_context = SubagentContext(
    shared_memory=child_context,  # Pass via existing field
    ...
)
```

### 8.2 Scoping Rules

The `for_subagent()` method creates a new context where:

- `goal` = The task being delegated
- `findings` = Empty (subagent starts fresh)
- `scratchpad` = Empty (no inherited state)

This prevents context pollution while providing task-specific framing.

### 8.3 Result Merging

After subagent completion:

```python
if result:
    self._context.add_finding(f"[{subagent_id}] {result[:100]}")
```

## 9. Export Format

### 9.1 Markdown Output

```python
def export(self, include_scratchpad: bool = False) -> str:
    parts = []
    if self.goal:
        parts.append(f"**Goal**: {self.goal}")
    if self.findings:
        parts.append("**Findings**:\n" + "\n".join(f"- {f}" for f in self.findings))
    if include_scratchpad and self.scratchpad:
        parts.append(f"**Scratchpad**: {json.dumps(self.scratchpad)}")
    return "\n\n".join(parts)
```

### 9.2 Prompt Injection

In graph nodes:

```python
context_summary = state.get("context_summary", "")
if context_summary:
    system_parts.append(f"\n## Context\n{context_summary}")
```

## 12. Implementation Notes

### 12.1 Simplifications

This RFC intentionally simplifies the cognitive architecture:

- No event-sourced reconstruction
- No EventStore dependency
- No complex projection machinery

CognitiveContext is designed as a lightweight, pragmatic solution for immediate context needs.

### 12.2 Future Extensions

- Semantic summarization of findings
- Context embedding for similarity search
- Multi-turn conversation threading

## 13. Conclusion

CognitiveContext provides a minimal, effective solution for maintaining agent state across TUI conversations and subagent interactions. By focusing on three essential fields (goal, findings, scratchpad) with optional memory integration, it delivers immediate value without the complexity of full event-sourcing.
