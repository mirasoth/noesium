## 1. Abstract

This RFC defines CognitiveContext, a minimal cognitive state model for maintaining context continuity across TUI conversations and subagent interactions. It provides a lightweight alternative to full event-sourced projections, using a 3-field model (goal, findings, scratchpad) with mandatory memory persistence and context-window-aware finding generation.

---

## 2. Scope and Non-Goals

### 2.1 Scope

This RFC defines:

* CognitiveContext data model and API
* Integration with NoeAgent's event loop
* Context scoping for subagent delegation
* Mandatory memory persistence integration
* Context window management and finding generation strategy

### 2.2 Non-Goals

This RFC does **not** define:

* Event-sourced state reconstruction (see RFC-1002 for projection model)
* Full cognitive architecture (deferred to future RFCs)
* LLM prompt engineering for context injection

---

## 3. Background & Motivation

NoeAgent lacks persistent cognitive state across TUI turns. When a user interacts with the agent over multiple exchanges, context from previous turns is lost. Additionally, when delegating to subagents, there's no structured way to pass relevant context.

**Problems solved by CognitiveContext**:

1. **Context Loss**: Findings from tool calls and planning are forgotten between turns
2. **Subagent Isolation**: Subagents operate without knowledge of parent agent's context
3. **Memory Bloat**: Full event-sourced projections are heavyweight for simple context needs

---

## 4. Design Principles

1. **Minimality**: Three fields capture essential cognitive state (goal, findings, scratchpad)
2. **FIFO Trimming**: Auto-trim findings to prevent unbounded growth
3. **Scoped Inheritance**: Subagent context inherits parent findings for continuity, with fresh scratchpad to prevent plan pollution
4. **Mandatory Persistence**: Memory integration is required for context continuity across turns and sessions
5. **Context Window Awareness**: Finding generation and export adapt to LLM token budgets

---

## 5. Data Model

### 5.1 CognitiveContext Fields

```python
class CognitiveContext(BaseModel):
    """Minimal cognitive state for context continuity."""
    
    memory_manager: Any
    """REQUIRED - Memory manager for persistence (ProviderMemoryManager)."""
    
    session_id: str
    """REQUIRED - Unique identifier for this session (used as persistence key)."""
    
    goal: str = ""
    """Current high-level goal or user intent."""
    
    findings: list[str] = Field(default_factory=list)
    """Recent discoveries from tool calls, observations, or reasoning."""
    
    scratchpad: dict[str, Any] = Field(default_factory=dict)
    """Arbitrary key-value storage for plans, intermediate results, etc."""
    
    max_findings: int = 8
    """Maximum findings to retain (FIFO eviction)."""
    
    auto_save: bool = True
    """Automatically save context on changes."""
```

### 5.2 Field Semantics

| Field | Purpose | Update Trigger | Persistence |
|-------|---------|----------------|-------------|
| `memory_manager` | Memory provider facade | Agent initialization | Required |
| `session_id` | Session identity | Agent initialization | Required |
| `goal` | User intent | Start of conversation turn | Per-session |
| `findings` | Tool/observation results | TOOL_END, SUBAGENT_END events | Auto-trimmed |
| `scratchpad` | Plans, intermediate state | PLAN_CREATED events | Per-session |

## 7. NoeAgent Integration

### 7.1 Initialization

```python
class NoeAgent:
    async def initialize(self, ...):
        self._context = CognitiveContext(
            memory_manager=self._memory_manager,
            session_id=self.config.session_id,
            max_findings=self.config.context_max_findings,
        )
        # Load previous context if resuming session
        if self.config.resume_session:
            await self._context.load()
```

### 7.2 Event Loop Integration

In `astream_progress()`:

```python
async def astream_progress(self, message: str, ...):
    # Set goal at turn start
    self._context.set_goal(message)
    
    # Enrich from memory if configured
    if self.config.context_auto_recall:
        await self._context.enrich(memory=self.memory_manager)
    
    # Process events (finding generation is async)
    async for event in self._process_turn(...):
        await self._update_context_from_event(event)
        yield event
    
    # Save context after each turn
    if self._context.auto_save:
        await self._context.save()
```

### 7.3 Event-to-Context Mapping

Events are processed via `extract_finding()` (see Section 13.3) rather than naive truncation:

| Event Type | Context Update |
|------------|----------------|
| `TOOL_END` | `add_finding(await extract_finding(tool_name, result))` |
| `SUBAGENT_END` | `add_finding(await extract_finding(subagent_id, detail))` |
| `PLAN_CREATED` | `set_scratchpad("current_plan", plan_snapshot)` |

## 8. Subagent Context Scoping

### 8.1 Context Delegation

When invoking a subagent:

```python
child_context = self._context.for_subagent(task=message)
subagent_context = SubagentContext(
    shared_memory=child_context,  # Pass via existing field
    ...
)
```

### 8.2 Scoping Rules

The `for_subagent()` method creates a new context where:

- `goal` = The task being delegated
- `findings` = Inherited from parent (subagent has parent context)
- `scratchpad` = Empty (no inherited state)

Inheriting findings provides subagents with essential background knowledge from the parent's research, preventing redundant work. The fresh scratchpad ensures the subagent's intermediate plans don't pollute the parent state.

### 8.3 Result Merging

After subagent completion:

```python
if result:
    self._context.add_finding(f"[{subagent_id}] {result[:100]}")
```

## 9. Export Format

### 9.1 Markdown Output

```python
def export(self, include_scratchpad: bool = False, max_tokens: int | None = None) -> str:
    parts = []
    if self.goal:
        parts.append(f"**Goal**: {self.goal}")
    if self.findings:
        parts.append("**Findings**:\n" + "\n".join(f"- {f}" for f in self.findings))
    if include_scratchpad and self.scratchpad:
        parts.append(f"**Scratchpad**: {json.dumps(self.scratchpad)}")
    result = "\n\n".join(parts)
    
    # Trim to token budget if specified (see Section 13.2)
    if max_tokens and _estimate_tokens(result) > max_tokens:
        result = _trim_to_budget(parts, max_tokens)
    return result
```

### 9.2 Prompt Injection

In graph nodes:

```python
context_summary = state.get("context_summary", "")
if context_summary:
    system_parts.append(f"\n## Context\n{context_summary}")
```

## 13. Context Window Management

### 13.1 Problem

LLM context windows are finite. Naively injecting all findings and scratchpad data into prompts can exceed token budgets, leading to truncation or degraded performance. Additionally, raw tool results are often verbose -- simple character truncation (e.g., `result[:80]`) loses critical information.

### 13.2 Token-Aware Export

`export()` accepts an optional `max_tokens` parameter. When the rendered output exceeds the budget, findings are trimmed oldest-first until within budget:

```python
def export(self, include_scratchpad=False, max_tokens: int | None = None) -> str
```

Token estimation uses a character-based heuristic: `len(text) // 4` (~4 chars per token). This avoids tokenizer dependencies while providing reasonable approximation.

### 13.3 Finding Generation Strategy

Findings are generated from raw tool/subagent results via `extract_finding()`, which supports three modes:

| Mode | When Used | Method | Cost |
|------|-----------|--------|------|
| `truncate` | Fallback | `f"{source}: {result[:max_length]}"` | Zero |
| `smart` | Default | Sentence-boundary aware truncation | Zero |
| `llm` | Optional | LLM summarization of long results | 1 LLM call |

```python
async def extract_finding(
    self,
    source: str,
    raw_result: str,
    *,
    llm: Any = None,
    mode: str = "smart",
    max_length: int = 200,
) -> str
```

**Smart mode** truncates at the nearest sentence boundary within `max_length`, preserving complete thoughts rather than cutting mid-sentence. Falls back to character truncation when no boundary is found.

**LLM mode** sends the raw result (capped at 4000 chars) to the LLM with a one-sentence summarization prompt. Falls back to smart mode on failure.

### 13.4 Finding Consolidation

When findings accumulate near `max_findings`, older findings can be consolidated via LLM summarization into a single `[summary]` entry, preserving information density while freeing slots for new findings:

```python
async def consolidate_findings(self, llm: Any, keep_recent: int = 3) -> None
```

This replaces `N` old findings with one `[summary] ...` finding, keeping the most recent `keep_recent` findings intact.

```python
    context_max_export_tokens: int | None = None   # Token budget for export
```

## 14. Conclusion

CognitiveContext provides a minimal, effective solution for maintaining agent state across TUI conversations and subagent interactions. By focusing on three essential fields (goal, findings, scratchpad) with mandatory memory persistence and context-window-aware finding generation, it delivers immediate value without the complexity of full event-sourcing.
