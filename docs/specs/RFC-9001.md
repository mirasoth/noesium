## 1. Abstract

This RFC defines the concrete architecture for Voyager, a personal coding assistant webserver. It specifies the component structure, data flow, API contracts, and integration patterns with NoeAgent and GitHub. The architecture follows a three-tier model: Frontend (SPA), Backend (FastAPI), and Execution (NoeAgent), with a clear separation of concerns between presentation, orchestration, and execution.

## 3. Background & Motivation

RFC-9000 established Voyager's design philosophy. This RFC translates those principles into a concrete architecture. The primary challenges are:

1. **Streaming**: Real-time progress from NoeAgent to the frontend
2. **State Management**: Task lifecycle and repository state
3. **GitHub Integration**: Clone, commit, and push operations
4. **Simplicity**: Minimal complexity for a single-user system

The architecture addresses these through:
* WebSocket-based streaming for real-time updates
* File-based state persistence for simplicity
* Git CLI integration for repository operations
* FastAPI for lightweight HTTP/WebSocket server

---

## 4. Design Principles

1. **Layered Architecture**: Clear separation between Frontend, Backend, and Execution
2. **Stateless Backend**: All state persisted to filesystem; server can restart without data loss
3. **Streaming First**: WebSocket for all long-running operations
4. **File-Based Storage**: No database required; JSON/JSONL for persistence
5. **Git as VCS**: All code operations through Git; Voyager doesn't implement VCS

### 7.2 WebSocket Protocol

**Connection**: `ws://localhost:8000/ws/tasks/{task_id}`

**Message Format**:

```typescript
// Server → Client: Progress Event
{
  "type": "progress",
  "event": {
    "type": "step.start" | "step.complete" | "tool.start" | "tool.end" | 
            "thinking" | "text.chunk" | "final.answer" | "error",
    "timestamp": "2026-03-04T12:00:00Z",
    "session_id": "...",
    "sequence": 1,
    "summary": "Step 1: Analyzing codebase structure",
    "detail": "...",
    "step_index": 0,
    "step_desc": "Analyze codebase structure",
    "tool_name": "bash",
    "tool_args": {...},
    "tool_result": "..."
  }
}

// Client → Server: Control
{
  "type": "control",
  "action": "cancel" | "pause" | "resume"
}
```

**Event Types** (mapped from NoeAgent `ProgressEventType`):

| Frontend Event | NoeAgent Event | Description |
|----------------|----------------|-------------|
| `session.start` | `SESSION_START` | Task execution started |
| `plan.created` | `PLAN_CREATED` | Execution plan generated |
| `step.start` | `STEP_START` | Step execution began |
| `step.complete` | `STEP_COMPLETE` | Step execution finished |
| `tool.start` | `TOOL_START` | Tool invocation started |
| `tool.end` | `TOOL_END` | Tool invocation finished |
| `thinking` | `THINKING` | Agent reasoning |
| `text.chunk` | `TEXT_CHUNK` | Text output chunk |
| `final.answer` | `FINAL_ANSWER` | Task completed |
| `error` | `ERROR` | Error occurred |

### 9.2 Task Orchestrator

```python
class TaskOrchestrator:
    """Manages task lifecycle and NoeAgent integration."""
    
    def __init__(
        self,
        state_manager: StateManager,
        github_client: GitHubClient,
        session_manager: SessionManager,
    ):
        self._state = state_manager
        self._github = github_client
        self._sessions = session_manager
    
    async def create_task(
        self,
        title: str,
        description: str,
        repository_id: str | None = None,
        branch: str | None = None,
    ) -> Task:
        """Create a new task."""
        task = Task(
            task_id=generate_id(),
            title=title,
            description=description,
            repository_id=repository_id,
            branch=branch,
            created_at=datetime.utcnow(),
            updated_at=datetime.utcnow(),
        )
        await self._state.save_task(task)
        return task
    
    async def execute_task(
        self,
        task_id: str,
        websocket: WebSocket,
    ) -> None:
        """Execute task with NoeAgent, streaming progress via WebSocket."""
        task = await self._state.get_task(task_id)
        
        # Prepare context
        context = await self._build_context(task)
        
        # Get or create NoeAgent instance
        agent = await self._sessions.get_agent(task.repository_id)
        
        # Stream progress events
        async for event in agent.astream_progress(task.description, context):
            # Forward to WebSocket
            await websocket.send_json({
                "type": "progress",
                "event": event.model_dump(),
            })
            
            # Update task state
            await self._update_task_from_event(task, event)
        
        # Mark task complete
        task.status = TaskStatus.COMPLETED
        task.completed_at = datetime.utcnow()
        await self._state.save_task(task)
```

### 9.3 GitHub Client

```python
class GitHubClient:
    """Git operations for repository management."""
    
    def __init__(self, workspace_root: Path):
        self._workspace = workspace_root
    
    async def clone(
        self,
        url: str,
        name: str,
    ) -> Path:
        """Clone repository to workspace."""
        repo_path = self._workspace / name
        result = await run_command(
            ["git", "clone", url, str(repo_path)],
        )
        return repo_path
    
    async def pull(self, repo_path: Path) -> None:
        """Pull latest changes."""
        await run_command(["git", "pull"], cwd=repo_path)
    
    async def create_branch(
        self,
        repo_path: Path,
        branch_name: str,
        base: str = "main",
    ) -> None:
        """Create and checkout new branch."""
        await run_command(
            ["git", "checkout", "-b", branch_name, base],
            cwd=repo_path,
        )
    
    async def commit(
        self,
        repo_path: Path,
        message: str,
    ) -> str:
        """Stage all changes and commit."""
        await run_command(["git", "add", "-A"], cwd=repo_path)
        await run_command(
            ["git", "commit", "-m", message],
            cwd=repo_path,
        )
        result = await run_command(
            ["git", "rev-parse", "HEAD"],
            cwd=repo_path,
        )
        return result.stdout.strip()
    
    async def push(
        self,
        repo_path: Path,
        branch: str,
    ) -> None:
        """Push branch to remote."""
        await run_command(
            ["git", "push", "-u", "origin", branch],
            cwd=repo_path,
        )
    
    async def get_diff(
        self,
        repo_path: Path,
        base: str = "HEAD~1",
    ) -> str:
        """Get diff of changes."""
        result = await run_command(
            ["git", "diff", base],
            cwd=repo_path,
        )
        return result.stdout
```

### 9.4 State Manager

```python
class StateManager:
    """File-based state persistence."""
    
    def __init__(self, data_root: Path):
        self._data_root = data_root
        self._tasks_dir = data_root / "tasks"
        self._tasks_dir.mkdir(parents=True, exist_ok=True)
    
    async def save_task(self, task: Task) -> None:
        """Save task to JSON file."""
        path = self._tasks_dir / f"{task.task_id}.json"
        async with aiofiles.open(path, "w") as f:
            await f.write(task.model_dump_json(indent=2))
    
    async def get_task(self, task_id: str) -> Task:
        """Load task from JSON file."""
        path = self._tasks_dir / f"{task_id}.json"
        async with aiofiles.open(path) as f:
            data = await f.read()
        return Task.model_validate_json(data)
    
    async def list_tasks(
        self,
        status: TaskStatus | None = None,
        limit: int = 50,
    ) -> list[Task]:
        """List tasks, optionally filtered by status."""
        tasks = []
        for path in sorted(
            self._tasks_dir.glob("task-*.json"),
            reverse=True,
        ):
            task = await self.get_task(path.stem)
            if status is None or task.status == status:
                tasks.append(task)
            if len(tasks) >= limit:
                break
        return tasks
    
    async def append_event(
        self,
        task_id: str,
        event: ProgressEvent,
    ) -> None:
        """Append event to task's event log."""
        path = self._tasks_dir / f"{task_id}-events.jsonl"
        async with aiofiles.open(path, "a") as f:
            await f.write(event.model_dump_json() + "\n")
```

## 10. NoeAgent Integration

### 10.1 Session Manager

```python
class SessionManager:
    """Manages NoeAgent instances per repository."""
    
    def __init__(
        self,
        state_manager: StateManager,
        github_client: GitHubClient,
    ):
        self._state = state_manager
        self._github = github_client
        self._agents: dict[str, NoeAgent] = {}
    
    async def get_agent(
        self,
        repository_id: str | None,
    ) -> NoeAgent:
        """Get or create NoeAgent for repository context."""
        if repository_id in self._agents:
            return self._agents[repository_id]
        
        repo = await self._state.get_repository(repository_id)
        
        config = NoeConfig(
            mode=NoeMode.AGENT,
            working_directory=str(repo.local_path),
            enabled_toolkits=[
                "bash",
                "file_edit",
                "python_executor",
                "document",
                "github",
            ],
            permissions=[
                "fs:read",
                "fs:write",
                "shell:execute",
                "net:outbound",
            ],
        )
        
        agent = NoeAgent(config)
        await agent.initialize()
        
        self._agents[repository_id] = agent
        return agent
    
    async def cleanup(self) -> None:
        """Cleanup all agent instances."""
        for agent in self._agents.values():
            # Cleanup subagents, etc.
            pass
        self._agents.clear()
```

### 10.2 Task Context Building

```python
async def _build_context(self, task: Task) -> dict[str, Any]:
    """Build execution context for NoeAgent."""
    context = {}
    
    if task.repository_id:
        repo = await self._state.get_repository(task.repository_id)
        context["repository"] = {
            "name": repo.name,
            "path": str(repo.local_path),
            "branch": task.branch or repo.default_branch,
        }
        
        # Include recent file changes
        recent_changes = await self._get_recent_changes(repo)
        context["recent_changes"] = recent_changes
    
    # Include related past tasks
    related_tasks = await self._find_related_tasks(task)
    if related_tasks:
        context["related_tasks"] = [
            {"task_id": t.task_id, "title": t.title}
            for t in related_tasks[:3]
        ]
    
    return context
```

3. **Access Control**: Restrict to trusted network

```bash
[Install]
WantedBy=multi-user.target
```

## 13. Security Considerations

### 13.1 Trust Model

Voyager operates with **full trust**:

* No authentication (single user)
* Full filesystem access within workspace
* Shell command execution via tools
* Network access via tools

**Constraint**: Only run Voyager on trusted machines.

### 13.2 GitHub Token

GitHub operations require a personal access token:

* Store in `~/.voyager/config.json` or `GITHUB_TOKEN` env var
* Token needs `repo` scope for private repositories
* Token is passed to git commands via credential helper

### 13.3 API Key Protection

LLM API keys are sensitive:

* Store in environment variables, not config files
* Never log or display API keys
* Rotate keys periodically

## 14. Error Handling

### 14.1 Task Failures

When a task fails:

1. Status set to `FAILED`
2. Error message stored in `final_answer`
3. Partial results preserved
4. User can retry with adjusted input

### 14.2 Repository Sync Failures

When sync fails:

1. Local state preserved
2. Error shown in UI
3. User can manually resolve conflicts
4. Retry after resolution

### 14.3 WebSocket Disconnects

On disconnect:

1. Task continues executing in background
2. Progress events logged to file
3. User can reconnect to view progress
4. Final results available on task detail page
