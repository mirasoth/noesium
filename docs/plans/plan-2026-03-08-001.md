# NoeAgent Code Structure Optimization Plan

## Context

This optimization aligns the noeagent code structure with RFC-1005 (Autonomous Architecture) and RFC-1009 (Layered Architecture) specifications. The recent reorganization introduced new directories (`capabilities/`, `graph/`, `kernel/`, `legacy/`, `session/`, `ui/`) but several issues remain:

1. **Broken import**: `kernel/__init__.py` imports `from .core import NoeAgent` but no `core.py` exists
2. **Code duplication**: `subagent_runtimes.py` exists at root level AND in `capabilities/subagents.py`
3. **Empty placeholder directories**: `session/` and `ui/` have no implementation code
4. **Incomplete Agent Kernel**: RFC-1005 defines Agent Kernel as one of five core autonomous components, but the implementation is incomplete

The goal is to fix these structural issues while maintaining backward compatibility and aligning with the autonomous architecture design.

---

## Implementation Plan

### Phase 1: Create Agent Kernel Implementation

The Agent Kernel is the reasoning engine that produces executable decisions (RFC-1005 Section 8). Currently, `CognitiveLoop` accepts a raw `NoeAgent` as its `agent_kernel` parameter, but should use a proper kernel abstraction.

**File: `/noeagent/src/noeagent/kernel/core.py` (NEW)**

Create an `AgentKernel` class that:
- Wraps `NoeAgent` to expose a `step(goal, context) -> Decision` interface
- Implements the reasoning, planning, and decision-making logic
- Provides a clean abstraction between the CognitiveLoop and NoeAgent

### Phase 2: Remove Code Duplication

**Action: DELETE `/noeagent/src/noeagent/subagent_runtimes.py`**

The root-level `subagent_runtimes.py` (448 lines) is NOT imported anywhere in the codebase. The canonical implementation is in `/noeagent/src/noeagent/capabilities/subagents.py` (564 lines), which is actively used by NoeAgent.

**Verification:** Grep confirms no imports of `noeagent.subagent_runtimes` anywhere.

### Phase 3: Remove Empty Placeholder Directories

**Actions:**
1. DELETE `/noeagent/src/noeagent/session/` directory (only contains empty `__init__.py`)
2. DELETE `/noeagent/src/noeagent/ui/` directory (only contains empty `__init__.py`)

**Rationale:**
- Session management is already handled by NoeAgent and `noesium.core.event`
- TUI is already fully implemented in `/noeagent/src/noeagent/tui.py`
- Empty directories violate YAGNI principle and create confusion
